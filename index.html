项目小记 | 关于实时邮件报警项目

背景

产品有泄露测试工位，为了减少报废，需要做一个预警系统，一旦数据组的偏差值大于某个预设阈值，则将邮件发出，报警邮件中配有图片，实时数据组，以及数据离散图示。

软件设计

1. 因为不是一条线，所以需要考虑到多线程

2. 数据采集使用的labview，采集一定量数据，并且在设定量的数据达到目标偏差时，将当前实时数据写入文件。

3. Python进行文件时间戳监控，一旦更新，则将数据读取

4. 考虑到labview统计方法与正常的统计方法有差异，需要使用python进行二次确认

5. 需要2个子程序，一个负责发邮件，一个负责生成图片

class my_thread(threading.Thread):

def __init__(self,threadID,name,path,receiver):
    threading.Thread.__init__(self)
    self.threadID = threadID
    self.name = name
    self.path = path
    self.receiver = receiver

def run(self):
    self.modifydateold = time.strftime('%Y-%m-%d %H:%M:%S',time.localtime(os.stat(self.path).st_mtime))
    while True:
      #  if not os.path.exists(self.path):

            self.modifydate = time.strftime('%Y-%m-%d %H:%M:%S',time.localtime(os.stat(self.path).st_mtime))

中位数

中位数查找

记录思路：

1. 始终需要确定2个数，处于中位，所以我们需要确认不同情况下的这两个数
2. 情况一：在[1，2]，[-1，0]，比如上面两个数列，数列1一个都不取，那第一个数数列2出的，为[0]，第二个数需要满足是数列1（最大那个）和数列2中的最小值，其中有分两种情况，一种是数列一样长，所以temp2有2种取值，最后如果temp1小于temp2，则可以直接返回中位数。根据奇偶的不同情况。
3. 情况二: 数列1取数字，从1个到n-1个，因为到了N个，这两个中位数的取值又不同，而这个情况是大多数的情况，需要这样来遍历，第一个数取两个数列最大的那个值为TEMP1，第二个数去后面一位的两个中最小的那个数，如果满足temp1小于temp2，则中位数找到。如果遍历完整个数列1，没有满足，那说明什么比如[1，3，5]，[-2，-1，0]，temp1为1，temp2为0，不成立，下一次，temp1为3，temp2为-1，还是不成立。说明temp2中的所有值都要比temp1小，而这种情况，在什么都不取得时候可以得到结果。
4. 情况三：遍历数列1的到最后一个时候，说明temp2目前为止的数都要比temp1的小，最后一个的时候，temp1还是2个中取最大的，temp2为只有nums2中取了。
